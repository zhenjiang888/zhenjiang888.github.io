To specify, design, implement, read, understand, document, appreciate, criticize, test, qualify, debug. maintain, adapt, port or improve programs, we need to understand the notations which serve to express their final form: programming languages.

Understanding programming and programming languages requires mastering the underlying theoretical models. This lecture explores the main methods of describing programming language syntax and semantics, with particular emphasis on abstract syntax, aximomatic semantics and denotational semantics.

This lecture will be useful to software engineering students as well as to computer science students. All mathmatical concepts will be discussed in a way with the emphasis on their applications to practical programming problems.
